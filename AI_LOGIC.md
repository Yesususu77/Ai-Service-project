# EditMuse! Analysis & Playback Logic Specification

본 문서는 **EditMuse!** 서비스의 핵심 엔진인 실시간 텍스트 분석 알고리즘과 사운드 매핑 로직의 상세 설계 사양을 기술합니다.

---

## 1. 지능형 분석 트리거 (Adaptive Analysis Trigger)
사용자의 집필 흐름을 방해하지 않으면서 문맥을 정확히 파악하기 위해 두 가지 독립적인 트리거를 운용합니다.

*   **문장 단위 감지 (Sentence-End Detection)**: `. ! ? …` 및 엔터 입력을 감지하여 문장이 완성된 시점에 분석을 실행합니다. (1초 쿨다운 적용)
*   **변화량 기반 디바운스 (Volumetric Debounce)**: 문장 부호가 없는 긴 문장이나 대량 편집 상황을 고려하여, **100자 이상의 변화** 감지 시 **1.5초 후** 분석을 실행합니다.

## 2. API 통신 및 데이터 가용성 전략 (Robust API Strategy)
네트워크 불안정성과 응답 지연(Latency) 상황에서도 서비스의 연속성을 보장합니다.

*   **레이스 컨디션 방지 (Race Condition Prevention)**: 각 요청에 고유한 `requestId`를 부여합니다. 이전 요청이 늦게 도착하더라도 현재 ID와 일치하지 않으면 폐기하여 항상 최신 분석 결과만 반영합니다.
*   **결함 허용 (Fault Tolerance)**: 5초 타임아웃 및 최대 2회의 점진적 재시도(Retry) 로직을 적용합니다. 최종 실패 시 `lastValidResult`를 반환하여 사운드 끊김을 방지합니다.

## 3. 스타일별 오디오 컨텍스트 (Dynamic Style Context)
선택된 장르 테마에 따라 AI의 판단 기준과 가용 SFX 리소스를 동적으로 분리하여 제공합니다.

| 스타일 (Style) | 사운드 아이덴티티 | 주요 효과음(SFX) 예시 |
| :--- | :--- | :--- |
| **Oriental** | 동양 전통 악기 (가야금, 대금 등) | 칼소리, 검기, 함성, 말발굽, 풍경소리 |
| **Orchestral** | 서양 클래식 및 오케스트레이션 | 마법이펙트, 폭발, 성문소리, 드래곤 |
| **Piano** | 감성적 어쿠스틱 및 피아노 선율 | 카페소음, 진동, 키보드, 자동차, 와인잔 |
| **Dark** | 다크 앰비언트 및 저음 드론 | 심장소리, 숨소리, 비명, 사이렌 |
| **Lofi** | 로파이 비트 및 힐링 사운드 | 식기소리, 요리소리, 고양이, 시냇물 |

## 4. 데이터 정제 및 후처리 (Strict Validation)
AI의 응답을 그대로 사용하지 않고, 서비스 규격에 맞게 2차 검증(Post-processing)을 거칩니다.

*   **감정 정규화 (Mood Normalization)**: 사전 정의된 **10종 감정 리스트** 이외의 값은 필터링하며, 유효값이 없을 경우 '평화'를 기본값으로 할당합니다.
*   **에너지 보정 (Energy Clamping)**: 분석된 에너지 수치를 1~5 범위로 정규화(Rounding & Clamping)하여 BGM 강도를 제어합니다.
*   **SFX 교차 검증**: 공용 목록과 해당 스타일 전용 목록에 포함된 키워드만 추출하며, 최대 3개로 제한합니다.

## 5. 재생 최적화 알고리즘 (Playback Optimization)
청각적 피로도를 낮추고 자연스러운 분위기 전환을 위해 적용된 알고리즘입니다.

*   **감정 안정화 버퍼 (Mood Stabilization Buffer)**: 단발적인 감정 변화에 BGM이 민감하게 반응하지 않도록 크기 3의 슬라이딩 윈도우 버퍼를 사용합니다. 버퍼 내 **동일 감정이 70% 이상**일 때만 실제 BGM을 교체합니다.
*   **초기 재생 로직 (Warm-up)**: 서비스 시작 후 첫 분석 시에는 버퍼 과정을 생략하고 즉시 사운드를 출력하여 초기 정적 구간을 제거합니다.
*   **글로벌 쿨다운 (SFX Cooldown)**: 동일한 효과음 키워드는 **30초의 재생 간격**을 강제하여 반복 재생으로 인한 소음을 방지합니다.

## 6. AI Interface Specification (API 설계)

분석 엔진과 시스템 간의 데이터 규격을 정의합니다. 모든 통신은 상태 유지와 순서 보장을 위해 설계된 파라미터를 포함합니다.

### [POST] /analyze
텍스트를 분석하여 감정, 에너지, 효과음 및 교정 제안을 반환합니다.

**Request Body**
| 필드명 | 타입 | 설명 |
| :--- | :--- | :--- |
| `text` | string | 분석할 소설 본문 (최근 500자 슬라이싱 적용) |
| `style` | string | 현재 선택된 음악 스타일 (예: `oriental`, `dark` 등) |
| `requestId` | number | 요청 순서 보장을 위한 고유 ID |

**Response Body (JSON)**
```json
{
  "mood": ["로맨틱"],      // 10종 고정값 중 선정
  "energy": 4,            // 1~5 정수
  "sfx": ["심장소리"],     // 스타일별 허용 리스트 내 추출
  "errors": [             // 맞춤법 및 문법 교정 제안
    {
      "type": "spell",
      "original": "바꼇다",
      "fix": "바뀌었다"
    }
  ],
  "requestId": 102        // 요청 시 전달받은 ID 그대로 반환
}
