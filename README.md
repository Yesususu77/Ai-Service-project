# EditMuse! AI System Architecture & Design

본 문서는 **EditMuse!**의 핵심인 AI 감정 분석 및 사운드 매핑 엔진의 설계 사양을 기술합니다. 본 시스템은 사용자의 집필 흐름을 실시간으로 추적하여 최적의 청각적 몰입감을 제공하는 것을 목표로 합니다.

---

## 1. AI 분석 엔진 및 프롬프트 설계 (Prompt Engineering)
사용자 텍스트를 분석하여 음악적 파라미터로 변환하기 위해 고도로 정제된 **Strict Prompting** 기술을 적용했습니다.

*   **Persona**: 한국 웹소설 및 장르 문학 전문 음악 감독 페르소나 부여.
*   **Style Context**: 스타일(Oriental, Orchestral, Piano, Dark, Lofi)별 악기 구성 및 효과음 가이드라인 동적 분리.
*   **Context Slicing**: 분석 효율성을 위해 입력 텍스트의 마지막 500자를 집중 분석하여 실시간성 확보.
*   **Output Format**: Pydantic 기반의 엄격한 JSON 스키마를 준수하여 시스템 안정성 극대화.

## 2. 감정 매핑 로직 (The 10-Mood System)
시스템의 일관성을 위해 감정 분석의 범위를 아래 10가지 핵심 감정으로 제한(**Strict Mapping**)합니다.

| 감정 | 설명 | 주요 악기 / 분위기 |
| :--- | :--- | :--- |
| **긴장** | 긴박한 대치, 위기 상황 | 빠른 현악기 피치카토, 긴박한 비트 |
| **로맨틱** | 고백, 설렘, 따뜻한 교감 | 부드러운 피아노, 플룻, 어쿠스틱 기타 |
| **슬픔** | 이별, 고립, 쓸쓸한 분위기 | 대금, 첼로 솔로, 낮은 패드 사운드 |
| **액션** | 전투, 추격, 박진감 | 거친 타악기, 브라스, 빠른 템포 |
| **평화** | 일상, 휴식, 안정된 상태 | 잔잔한 자연음, 앰비언트 사운드 |
| **기타** | 신비, 공포, 희망, 분노, 코믹 | 각 감정에 특화된 테마 사운드 매핑 |

## 3. 지능형 분석 트리거 (Adaptive Analysis Trigger)
사용자의 집필 리듬을 방해하지 않으면서 최적의 시점에 분석을 실행하기 위한 이중 트리거 전략입니다.

*   **문장 단위 감지 (Sentence-End)**: `. ! ? …` 및 엔터 입력 시 문맥이 완성된 것으로 판단하여 1초 쿨다운 후 분석 실행.
*   **변화량 기반 디바운스 (Volumetric)**: 문장 부호가 없는 상황에서도 100자 이상의 변화 발생 시 1.5초 디바운스 적용.

## 4. 시스템 안정화 및 신뢰성 기술 (Reliability & UX)
단순 분석을 넘어 실시간 오디오 서비스의 고질적인 문제를 알고리즘으로 해결했습니다.

*   **Request ID Tracking**: 모든 요청에 고유 ID를 부여하여, 레이스 컨디션 발생 시 가장 최신 데이터만 반영하도록 순서 보장.
*   **Mood Stabilization Buffer**: 크기 3의 슬라이딩 윈도우 버퍼를 운용, 동일 감정이 **70% 이상**일 때만 BGM을 전환하여 급격한 음악 변화 방지.
*   **SFX Global Cooldown**: 동일한 효과음 키워드에 대해 **30초의 쿨다운**을 강제하여 반복 재생으로 인한 청각적 피로도 최소화.
*   **Fail-Safe**: API 실패 시 `lastValidResult`를 활용한 폴백(Fallback) 메커니즘을 통해 끊김 없는 서비스 제공.

## 5. 데이터 인터페이스 규격 (API Specification)
프론트엔드와 AI 엔진 간의 데이터 교환을 위한 표준 규격입니다.
```json
{
  "mood": ["로맨틱"],      // 정의된 10종 고정값 내 선정
  "energy": 4,            // 1~5 정수 기반 사운드 강도
  "sfx": ["심장소리"],     // 스타일별 허용 리스트 내 최대 3개 추출
  "errors": [             // 실시간 맞춤법 및 문법 교정 제안
    { "type": "spell", "original": "바꼇다", "fix": "바뀌었다" }
  ],
  "requestId": 102        // 비동기 요청 순서 검증용 ID
}
'''

## 6. Core Files
- `app/core/prompts.py`: 마스터 시스템 프롬프트 관리
- `app/core/config.py`: 시스템 전역 파라미터 및 상수 설정
- `app/core/utils.py`: AI 응답 데이터 검증 및 클램핑 로직
- 'app/core/analyzer.js': 분석 엔진 본체. 트리거 제어, API 통신, 버퍼링 및 후처리 로직 전담.
